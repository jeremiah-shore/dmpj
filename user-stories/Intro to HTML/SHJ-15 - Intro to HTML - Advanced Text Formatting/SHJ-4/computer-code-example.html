Encapsulation in Java

The following excerpt demonstrates a few of the ideas behind the principle of encapsution in object-oriented design, specifically, in Java.


	public class Example {
		private int value;
		private static final int INCREMENT_AMOUNT = 5;

		private Example() {	}

		public Example(int value) {
			this.value = value;
		}

		public int getValue() {
			return value;
		}

		public int incrementValue() {
			value += INCREMENT_AMOUNT;
		}
	}

encapsulated constructor

The default constructor is explicitly declared, but no bahavior is defined, and it is also set to private. The effect of this is that no other code can create an instance of Example through its default constructor; the public constructor defined here that accepts an integer is the only valid way to create an Example instance.

In other words, we are encapsulating the means by which an Example can be created, giving the author full control of this behavior, and assurance that a value is always defined by a user of this API upon instance creation.

encapsulated member fields

The member variable integer value is also set to private. We are using public methods to retrieve and modify this value so that it cannot be retrieved or reassigned directly. This not only prevents unwanted or unexpected behavior, but allows us to more precisely define the logic related to this class and its attributes.

A commonly used example of encapsulation uses private member fields and public "getters and setters". However, we may not always want to define our code in this way; we may want to exercise precise control over our variables by defining how they are retrieved, or in this case, modified. Notice the public method incrementValue() and its use of the constant INCREMENT_AMOUNT. In this example, the value can never be reassigned or updated to a user defined value at any point after instance creation. Instead, it must instead always be increased by this predefined increment.

This function definition encapsulates the update behavior of value from users of this API, more clearly defining how they are intended to interact with our Example class.
